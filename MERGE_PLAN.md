## 1. Strategy Overview

### Merge Principles

1. **No data loss** — every document from both archives must be preserved
2. **Archive B is the baseline** — governance and architecture documents from B are the latest versions
3. **Archive A adds source code and product documentation** — scan platform, engines, pipeline, strategy, execution
4. **Overlapping files are merged** — combine content from both sources, preferring the more complete version
5. **Directory structure is reorganized** — unified structure per OPS-004 spec
6. **Historical documents are archived** — not deleted, moved to docs/archive/
