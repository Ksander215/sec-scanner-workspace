# First Value Experience — UX Audit

## Текущий сценарий пользователя

```text
Landing Page
    ↓ (скролл до #auth-panel)
Регистрация / Вход
    ↓ (window.location.reload — хак)
Dashboard (Scan tab)
    ↓ (ввести URL, нажать Start)
Ожидание сканирования (4-8 сек)
    ↓ (переключиться на History tab вручную)
История сканирований
    ↓ (нажать на строку)
Scan Detail Modal (vulnerabilities)
```

## Time to First Value

| Этап | Время | Проблема |
|------|-------|----------|
| Регистрация | ~30 сек | Двойной submit (register + signIn) |
| Post-login reload | ~3 сек | `window.location.reload()` хак |
| Понимание интерфейса | ~60 сек | Нет онбординга, "How it works" занимает 50% экрана |
| Ввод URL | ~15 сек | OK |
| Ожидание скана | 4-8 сек | Нет индикации в текущей вкладке |
| Переключение на History | ~10 сек | Пользователь не знает, что нужно переключиться |
| Открытие результатов | ~5 сек | |
| **Итого** | **~2-3 мин** | **При идеальном сценарии** |

## Найденные проблемы (по приоритету)

### Критические

1. **Нет Security State / Explainability** — главный продукт (Security Score, рекомендации, причины) полностью отсутствует в UI. Пользователь видит только "сырые" findings.

2. **Нет целевой модели (Target)** — пользователь сканирует URL, но нет понятия "цель мониторинга". Каждый скан — изолированное событие, а не часть непрерывного процесса.

3. **Нет перехода от скана к действиям** — после скана пользователь видит таблицу уязвимостей, но не видит:
   - Что это значит для безопасности в целом?
   - Что исправить в первую очередь?
   - Какой эффект даст исправление?

4. **"How it works" блокирует 50% экрана** — на вкладке Scan, половина пространства занята инструкцией, которая нужна только при первом использовании.

### Высокие

5. **Billing tab — мёртвый конец** — все кнопки disabled, нет функционала.

6. **Нет empty states** — "No scans yet" — просто текст без визуала и без CTA.

7. **SPA без истории** — нет deep linking, кнопка "назад" ломает навигацию.

8. **Пост-login reload** — `setTimeout(() => window.location.reload(), 200)` — ненадёжный хак.

### Средние

9. **Нет команды "Повторить скан"** — нужно заново вводить URL.

10. **История: нет "последнего скана" на первом экране** — пользователь не видит текущее состояние без лишних кликов.

11. **Непоследовательный i18n** — ~50% строк захардкожены.

12. **Stale text "refreshing every 2s"** — фактически 5 секунд.

## Рекомендации для TASK-007

### Что делаем сейчас (Sprint 2)

1. **Новая вкладка "Security" в Dashboard** — с Security Score, Explainability, рекомендациями. Это — первое, что видит пользователь после входа (вместо Scan tab).

2. **Mock-данные** — показываем Security State с реальными доменными модулями на демо-данных. Это демонстрирует ценность продукта до первого скана.

3. **Action-Oriented UX** — каждый блок заканчивается конкретным действием:
   - "Запустить первый скан" (empty state)
   - "Исправить Critical: XSS" (рекомендация)
   - "Повторить сканирование" (stale data)

4. **Empty States** — 5 состояний с иллюстрациями и CTA:
   - Нет данных (первый визит)
   - Сканирование в процессе
   - Идеальное состояние
   - Устаревшие данные
   - Нет данных для сравнения

### Что НЕ делаем сейчас

- Не рефакторим SPA → MPA (масштабный рефакторинг)
- Не меняем Security State Engine
- Не меняем Explainability Layer
- Не добавляем Workspace/Target в БД (это отдельная задача)
- Не внедряем AI
- Не чиним Billing (отдельная задача)

## Ожидаемый эффект

| Метрика | Текущее | Целевое |
|---------|---------|---------|
| Time to First Value | 2-3 мин | < 30 сек (демо-данные) |
| Понимание состояния | Нет | Security Score + объяснение |
| Следующее действие | Не понятно | Явная рекомендация |
| Пустые состояния | Текст | Визуал + CTA |
| Retention trigger | Отсутствует | "Исправьте это, чтобы получить +15 очков" |
