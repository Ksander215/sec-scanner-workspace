# STYLE_GUIDE.md — Product Style Guide

> EP-001 BLOCK 13. Единый стиль продукта: тон, термины, цвета, компоненты.

---

## Тон общения

### Принципы
1. **Дружелюбный, но профессиональный** — не формальный, но и не панибратский
2. **Бизнес-язык** — руководитель понимает, инженер находит детали
3. **Действенный** — каждая страница предлагает что сделать
4. **Честный** — лучше "не знаю" чем fake данные

### Примеры

| Ситуация | ❌ Плохо | ✅ Хорошо |
|----------|---------|-----------|
| Пустой список | "Нет данных" | "Пока нет сканирований. Запустите первое →" |
| Ошибка | "Error 500" | "Не удалось загрузить. Попробуйте ещё раз или обратитесь в поддержку." |
| Success | "Готово!" | "Сканирование завершено. Найдено 3 критические уязвимости. Посмотреть →" |
| Warning | "Внимание!" | "Истекает SSL сертификат через 7 дней. Обновить →" |
| Loading | "Loading..." | "Проверяем безопасность... Это займёт 2 минуты" |

|-------------------------------------|-------------------------|

| AIO | (не показывать пользователю) |

| Product Readiness | (не показывать пользователю) |

### Обращение к пользователю
- На "вы" (не на "ты")
- Без имени (пока нет персонализации)
- Без emoji в основном UI (только в AI Assistant приветствии)

## Цвета

### Основная палитра

| Цвет | HEX | Использование |
|------|-----|---------------|
| Accent (violet) | `#8b5cf6` | Primary actions, AI Assistant |
| Success (emerald) | `#10b981` | Позитивные статусы, success messages |
| Warning (amber) | `#f59e0b` | Предупреждения, partial статусы |
| Danger (red) | `#ef4444` | Критические ошибки, broken статусы |
| Info (blue) | `#3b82f6` | Информационные сообщения |
| Background | `#0a0a0a` (dark) / `#ffffff` (light) | Фон |
| Surface | `#141414` (dark) / `#f8f8f8` (light) | Карточки |
| Border | `#262626` (dark) / `#e5e5e5` (light) | Границы |

### Принципы использования
- **Accent** — только для primary CTA ("Запустить сканирование", "Подключить")
- **Success** — после реального успешного действия (не для mock!)
- **Danger** — для критических проблем, требующих немедленного действия
- Не использовать красный для некритических warning

## AI Voice

### Принципы
- AI Assistant говорит от первого лица ("я", "мне")
- Не использует технические термины без объяснения
- Предлагает действия, не только информацию
- Помнит контекст (роль, историю, предпочтения)

### Примеры

| Ситуация | Ответ AI |
|----------|----------|
| Первый вход | "Здравствуйте! Я ваш AI Assistant. Помогу проверить безопасность. С чего начнём?" |
| После сканирования | "Сканирование завершено. Нашёл 3 критические проблемы. Рекомендую начать с обновления SSL сертификата — это займёт 15 минут." |
| Пустой дашборд | "Пока нет данных. Запустите первое сканирование — это займёт 2 минуты. Я подскажу что делать дальше." |
| Ошибка | "Не удалось завершить сканирование. Это может быть из-за сетевых ограничений. Попробовать ещё раз?" |

## Компоненты

### Карточки

```
┌─────────────────────────────┐
│ Icon  Title              ✓  │
│       Subtitle              │
│ ─────────────────────────── │
│ Content...                  │
│ ─────────────────────────── │
│ Action button →             │
└─────────────────────────────┘
```

- Скругление: `rounded-xl`
- Тень: только при hover
- Border: тонкий, `border-border`

### Кнопки

| Тип | Стиль | Использование |
|-----|-------|---------------|
| Primary | `bg-accent text-white` | Главная CTA на странице (1 на страницу) |
| Secondary | `bg-surface border-border` | Альтернативные действия |
| Ghost | `text-muted hover:text-foreground` | Третичные действия |
| Danger | `bg-red-500 text-white` | Удаление, необратимые действия |

### Empty States

│                             │

│                             │

## Принципы UI

1. **Один primary CTA на страницу** — пользователь должен знать что главное
2. **Прогрессивное раскрытие** — сначала суть, детали по клику
3. **Empty state > Fake data** — лучше пусто, чем ненастоящее
4. **Действие > Информация** — каждая страница предлагает что сделать
5. **AI Assistant всегда доступен** — плавающая кнопка в правом нижнем углу
6. **Тёмная тема по умолчанию** — для security продукта это естественно
