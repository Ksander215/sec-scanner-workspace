# ADR-014: Feature Flags for Platform Modules

**Status:** Accepted

**Date:** 2026-07-15

## Context

The Security Intelligence Platform has 20+ infrastructure modules (distributed pipeline, event bus, saga, AI layer, SSO, SIEM, ticketing, CMDB, secrets, threat intel, detection, attack simulation, data lake, analytics, cloud connectors, etc.). Not all deployments need all modules. Requirements include:

- Enable/disable modules at deployment time without code changes
- Granular feature control within modules (e.g., enable AI layer but disable specific AI features)
- Runtime feature flag updates without restart
- Default configuration that works for common deployments
- Clear module dependency tracking

## Decision

We implemented a **feature flag system** in `Platform` (`src/infrastructure/platform/engine.ts`) using a flat key-value map:

```typescript
features: {
  'distributed-pipeline': true,
  'event-bus': true,
  'saga': true,
  'scheduler': true,
  'cluster-coordination': false,
  'multi-tenancy': true,
  'streaming': true,
  'ai-layer': false,
  'ai-risk-assistant': false,
  'ai-remediation': false,
  'ai-threat-hunting': false,
  'ai-copilot': false,
  'sso': false,
  'siem': false,
  'ticketing': false,
  'notification': true,
  'cmdb': false,
  'secrets': false,
  'threat-intel': true,
  'detection-engineering': true,
  'attack-simulation': false,
  'data-lake': false,
  'analytics': true,
  'dashboards': true,
  'compliance': false,
  'cloud-aws': false,
  'cloud-azure': false,
  'cloud-gcp': false,
  'cloud-k8s': false,
}
```

Feature flags are set at platform initialization and can be updated at runtime via `setFeature(feature, enabled)`.

### Module Initialization

The platform initializes modules based on their configuration:

```typescript
modules: {
  eventBus: { enabled: true },
  ai: { enabled: false, provider: 'openai' },
  enterprise: { sso: false, siem: false, ticketing: false, notification: true, ... },
  ...
}
```

Modules are initialized in a defined order: eventBus → multiTenancy → distributedPipeline → saga → scheduler → streaming → threatIntel → detection → analytics → dataLake → cloud → ai → enterprise.

## Consequences

### Positive

- **Incremental adoption** — New deployments start with core features enabled and enterprise features disabled. Features are enabled as needed.
- **Runtime toggling** — `setFeature()` enables enabling/disabling features without process restart. Useful for incident response (disable AI layer during LLM provider outage).
- **Default-off security** — Enterprise features (SSO, SIEM, secrets) are disabled by default, reducing the attack surface for basic deployments.
- **Clear feature inventory** — The flat map provides a complete list of platform capabilities and their status.

### Negative

- **No feature dependency graph** — Feature flags are independent. Enabling `ai-threat-hunting` without `ai-layer` is invalid but not enforced at the flag level. The module initialization logic must handle this.
- **No percentage rollout** — Feature flags are boolean (on/off). Gradual rollout (e.g., enable for 10% of tenants) is not supported.
- **No per-tenant features (at flag level)** — Feature flags are global. Per-tenant feature gating is handled by `TenantConfig.features`, creating two feature control mechanisms that must be kept consistent.
- **No audit trail** — `setFeature()` changes are not logged. Runtime flag changes are invisible in the audit log.

## Alternatives Considered

| Alternative | Why Rejected |
|------------|-------------|
| **LaunchDarkly / Unleash** | Production-grade feature flag services with A/B testing, percentage rollouts, and audit trails. Adds external dependency and cost. Suitable for SaaS but not for air-gapped deployments. |
| **Environment variables only** | Simple but requires process restart. No runtime updates. No per-module granularity. |
| **Config file only** | Requires file reload. No runtime updates. Harder to manage in containerized environments. |
| **No feature flags** | All modules always enabled. Wastes resources on unused modules and increases attack surface. |
