## Общие принципы

### Тон
- Профессиональный, но тёплый
- Уважительный к пользователю
- Без паники и страха
- Без снисходительности

### Язык
- Русский — основной язык интерфейса
- Английский — переключается в TopBar
- Технические термины — с пояснениями (TermTooltip)
- Все тексты проходят Product Review

## Цветовая система

### Основные цвета

| Назначение | Цвет | Класс Tailwind | Пример |
|-----------|------|---------------|--------|
| Акцент | Violet | `violet-500` / `text-accent` | Кнопки, ссылки, активные элементы |
| Успех | Green | `green-500` / `text-green-500` | Завершено, безопасно |
| Предупреждение | Amber | `amber-500` / `text-amber-500` | Обратить внимание |
| Опасность | Red | `red-500` / `text-red-500` | Критические проблемы |
| Информация | Blue | `blue-500` / `text-blue-500` | Справка, подсказки |

### Тёмная тема
- Фон: `bg-surface` (semi-transparent, backdrop-blur)
- Текст: `text-foreground` (автоматически адаптируется)
- Бордеры: `border-border`
- Акцент остаётся violet

### Правило использования
- Красный — ТОЛЬКО для реальных критических проблем
- Жёлтый — для предупреждений, которые требуют внимания
- Зелёный — для подтверждений и прогресса
- Violet — для действий, навигации, акцентов

## Компоненты

### Кнопки

**Primary** (основное действие):
- `bg-gradient-to-br from-violet-500 to-fuchsia-500`
- `text-white`
- `shadow-lg shadow-violet-500/25`
- Hover: `shadow-violet-500/40`

**Secondary** (дополнительное действие):
- `bg-surface border border-border`
- `text-foreground`
- Hover: `bg-accent/10`

**Ghost** (третьестепенное):
- `text-foreground/60`
- Hover: `text-foreground bg-foreground/5`

### Карточки
- `bg-surface border border-border rounded-xl`
- Padding: `p-4` или `p-6`
- Hover: `hover:border-accent/30 hover:shadow-accent/5`
- Transition: `transition-all duration-200`

### Бейджи
- Маленькие: `px-2 py-0.5 text-[10px] font-medium`
- Средние: `px-2.5 py-1 text-xs font-medium`
- Формат: `bg-{color}-500/10 text-{color}-600 dark:text-{color}-400 border border-{color}-500/20`

### Тултипы
- Использовать TermTooltip для технических терминов
- Появляются по hover
- Содержат простое объяснение на языке бизнеса

## Анимации

### Принципы
- Цель анимации — помочь пользователю, не развлечь
- Быстрые переходы (150-300ms)
- Естественные easing (ease-out для появления, ease-in для исчезновения)
- Respect `prefers-reduced-motion`

### Framer Motion
```typescript
// Появление
initial={{ opacity: 0, y: 10 }}
animate={{ opacity: 1, y: 0 }}
transition={{ duration: 0.2 }}

// Исчезновение
exit={{ opacity: 0, y: -10 }}
transition={{ duration: 0.15 }}
```

### Запрещено
- Бесконечные анимации без паузы
- Мерцание
- Анимации, которые невозможно отключить
- Анимации дольше 500ms для простых переходов

## Уведомления

### Стиль
- Минималистичный
- Современный
- «Дорогой» (ощущение премиум-системы)
- Как системные уведомления в Solo Leveling (без копирования)

### Структура
```
┌─────────────────────────────────┐
│ AIS ●        Тип    [×]         │
│                                 │
│ Заголовок уведомления           │
│ Описание что произошло          │
│                                 │
│ ████████████░░░  (timer bar)    │
└─────────────────────────────────┘
```

### Время жизни
- Адаптивное (см. AIS_SPECIFICATION.md)
- Минимум: 3 сек
- Максимум: 12 сек
- Ручное закрытие — всегда доступно

## Адаптивность

### Breakpoints
| Размер | Ширина | Устройство |
|--------|--------|-----------|
| sm | 640px | Телефон горизонтально |
| md | 768px | Планшет |
| lg | 1024px | Ноутбук |
| xl | 1280px | Десктоп |
| 2xl | 1536px | Широкий десктоп |

### Правила
- Sidebar скрывается на < md (бургер-меню)
- Карточки — одна колонка на мобильном, две на планшете, три на десктопе
- Графы (KG, Attack Paths) — минимальный размер для интерактивности
- AIS панель — полноэкранная на мобильном, 384px на десктопе

## Пустые состояния

Когда данных нет, показывать:
1. Иконку (крупная, приглушённая)
2. Заголовок («Нет данных» / «Ещё нет сканирований»)
3. Описание («Запустите первое сканирование, чтобы увидеть результаты»)
4. Кнопку действия («Запустить сканирование»)

Запрещено:
- Пустые белые пространства
- Сообщения типа «Данные не найдены» без предложения действия
- Фейковые данные вместо пустого состояния
