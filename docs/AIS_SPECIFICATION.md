# AIS_SPECIFICATION.md — Спецификация Adaptive Intelligence System

> AIS — это не чат-бот. Не FAQ. Не виджет.  
> AIS — это операционная система взаимодействия с платформой.

---

## Назначение

AIS превращает платформу из набора инструментов в интеллектуального сопровождающего. Пользователь не ищет помощь — помощь находит его сама.

AIS:
- Понимает, где находится пользователь
- Понимает, что он делает
- Понимает, что ему нужно прямо сейчас
- Предлагает помощь тогда, когда она действительно нужна
- Не раздражает

## Модуль 1: Adaptive Memory Engine (memory.ts)

### Назначение
Хранит профиль поведения пользователя в localStorage.

### Что хранит
- **Скорость чтения** — как быстро пользователь закрывает уведомления
- **Скорость навигации** — как быстро переходит между страницами
- **Любимые разделы** — какие страницы посещает чаще всего
- **Открытые советы** — какие советы открывал
- **Игнорированные советы** — какие закрывал без чтения
- **Частые действия** — что запускает чаще всего
- **Ежедневные действия** — чем пользуется ежедневно
- **Оценка роли** — CEO, DevOps, инженер, аналитик
- **Цели** — персональные цели пользователя
- **Метрики** — числовые показатели прогресса

### API
```typescript
interface AdaptiveMemory {
  recordAction(action: UserAction): void;
  getProfile(): UserProfile;
  getRole(): UserRole;
  getDismissedTips(): string[];
  getFavoriteSections(): string[];
  getReadingSpeed(): 'fast' | 'normal' | 'slow';
  getNavigationSpeed(): 'fast' | 'normal' | 'slow';
}
```

## Модуль 2: Sound Identity (sound.ts)

### Назначение
Фирменный звук AIS. Очень короткий, уникальный, не раздражающий.

### Характеристики
- Длительность: 100-200 мс
- Не похож на Telegram, Windows, iOS
- Реализация: Web Audio API
- 6 типов звуков: notification, success, error, complete, tip, achievement

### Настройки
- Включить / выключить (в Settings — не реализовано)

### API
```typescript
interface SoundEngine {
  play(type: SoundType): void;
  setEnabled(enabled: boolean): void;
  isEnabled(): boolean;
}
type SoundType = 'notification' | 'success' | 'error' | 'complete' | 'tip' | 'achievement';
```

## Модуль 3: Confidence Engine (confidence.ts)

### Назначение
Оценка уверенности платформы в своих рекомендациях. От 0 до 100.

### Уровни
| Уровень | Диапазон | Цвет | Значение |
|---------|----------|------|----------|
| Very Low | 0-20 | Красный | Мало данных, низкая уверенность |
| Low | 21-40 | Оранжевый | Ограниченные данные |
| Moderate | 41-60 | Жёлтый | Достаточно данных для общих выводов |
| High | 61-80 | Зелёный | Хорошая уверенность |
| Very High | 81-100 | Ярко-зелёный | Высокая уверенность в рекомендациях |

### Факторы
- Количество выполненных сканирований
- Количество найденных проблем
- Время с последнего сканирования
- Покрытие инфраструктуры
- Полнота данных

### Нарратив по ролям
- **CEO**: «Уровень контроля вашей инфраструктуры заметно вырос»
- **DevOps**: «Обнаружено 4 изменения. Посмотрите рекомендации»
- **Инженер**: «Есть 3 критических уязвимости, требующих внимания»
- **Аналитик**: «Есть изменения в общей картине рисков»

### API
```typescript
interface ConfidenceEngine {
  calculate(): ConfidenceResult;
  getLevel(): ConfidenceLevel;
  getNarrative(role: UserRole): string;
}
```

## Модуль 4: Context Predictor (context-predictor.ts)

### Назначение
Предсказывает, что нужно пользователю, на основе текущей страницы и профиля.

### Правила предсказания (7 правил)

| Страница | Подсказка |
|----------|-----------|
| Marketplace | «Не знаете, что выбрать? Я помогу подобрать инструменты» |
| Scanner | «Готовы проверить инфраструктуру? Начнём» |
| Reports | «Отчёт уже готов. Хотите отправить руководителю?» |
| Knowledge Graph | «Посмотрите, как связаны найденные проблемы» |
| Attack Paths | «Самое опасное место уже найдено» |
| Dashboard | «Сегодня ваша инфраструктура выглядит стабильнее, чем вчера» |
| Settings | (Тихий режим — минимальные подсказки) |

### API
```typescript
interface ContextPredictor {
  predict(path: string, profile: UserProfile): AISPrediction | null;
}
```

## Модуль 5: useAIS Hook (useAIS.ts)

### Назначение
React hook, объединяющий все AIS-подсистемы в единый интерфейс.

### Возвращаемые данные
```typescript
interface AISState {
  role: UserRole;                    // Текущая роль
  confidence: ConfidenceResult;      // Оценка уверенности
  goals: Goal[];                     // Персональные цели
  prediction: AISPrediction | null;  // Текущее предсказание
  readingSpeed: ReadingSpeed;        // Скорость чтения
  navSpeed: NavigationSpeed;         // Скорость навигации
}
```

## AIS Assistant (AISAssistant.tsx)

### Визуальный интерфейс

#### Плавающая кнопка
- Позиция: fixed, правый нижний угол
- Размер: 48×48px (w-12 h-12)
- Стиль: градиент violet-500 → fuchsia-500
- Иконка: Sparkles (✨)
- Тень: violet glow
- Анимация: hover scale + shadow pulse

#### Боковая панель
- Позиция: fixed, правая сторона, полноэкранная
- Ширина: 384px (w-96)
- Фон: backdrop-blur-xl, полупрозрачный
- 3 вкладки с иконками

### Вкладки

#### 1. Навигация (Compass icon)
Показывает контекстную подсказку для текущей страницы. Контент меняется при переходе между страницами.

#### 2. Цель (Target icon)
Показывает персональную цель пользователя с прогресс-баром. Цель адаптируется под роль.

#### 3. Уверенность (Shield icon)
Показывает оценку уверенности платформы + Trust Builder (факторы, влияющие на оценку).

## Solo Notification (SoloNotification.tsx)

### Визуальный стиль
- Минималистичный, современный, «дорогой»
- Ощущение «живой системы»
- Как системные уведомления в Solo Leveling (без копирования дизайна)

### Анимация появления
- Лёгкое появление (opacity)
- Небольшой масштаб (scale)
- Свечение (glow)
- Лёгкая вибрация карточки
- Плавное исчезновение

### 6 типов уведомлений
1. **info** — Информационное
2. **success** — Успех
3. **warning** — Предупреждение
4. **error** — Ошибка (мягко: «Есть возможность улучшить»)
5. **tip** — Совет
6. **achievement** — Достижение

### Адаптивное время жизни

| Поведение пользователя | Время жизни | Направление |
|----------------------|-------------|-------------|
| Быстро закрывает | 6 → 5 → 4 → 3 сек | Уменьшается |
| Медленно читает | 6 → 8 → 10 → 12 сек | Увеличивается |
| Всегда читает полностью | Не закрывается автоматически | Режим «Закрыть вручную» |
| Игнорирует подряд | Реже показываются | Снижение частоты |

### Правила показа
1. Никогда не показывать одно и то же сообщение подряд
2. Если пользователь проигнорировал совет — не повторять, показать новый
3. Если закрыл несколько раз подряд — уменьшить количество подсказок
4. Если всегда открывает — увеличить

## Психология AIS

### Запрещённые слова (если ситуация не критическая)
- ❌ «Ошибка»
- ❌ «Опасность»
- ❌ «Провал»
- ❌ «Критическая ошибка»
- ❌ «Срочно!»

### Рекомендуемые формулировки
- ✅ «Есть возможность улучшить»
- ✅ «Рекомендуем обратить внимание»
- ✅ «Следующий шаг поможет повысить уровень контроля»
- ✅ «Обнаружено 4 изменения — посмотрите рекомендации»

### «Вау»-сообщения
Особые сообщения, которые вызывают улыбку, а не ощущение рекламы:
- «За последние две недели уровень контроля вашей инфраструктуры заметно вырос»
- «Вы уже автоматизировали 61% процессов проверки. Продолжим?»
- «Сегодня вы устранили 3 потенциальные проблемы. Хорошая работа»

## Режимы работы AIS

| Режим | Описание |
|-------|----------|
| Автоматически скрывать | Уведомления исчезают по таймеру |
| Показывать дольше | Увеличенный таймер (для медленных читателей) |
| Показывать всегда | Режим «Закрыть вручную» |
| Без звука | Звуковые уведомления выключены |
| Компактный режим | Минимальная информация |
| Подробный режим | Расширенные объяснения |

### Частота подсказок
| Режим | Частота |
|-------|---------|
| Редко | 1 подсказка на 3+ действия |
| Нормально | 1 подсказка на 1-2 действия |
| Часто | Подсказка после каждого действия |
