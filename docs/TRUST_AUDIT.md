# TRUST_AUDIT.md — Аудит доверия пользователя

> EP-001 BLOCK 11. Каждое место, где пользователь может подумать "это ненастоящее".
> Приоритет: P0 (критично для коммерции), P1 (важно), P2 (улучшить).

---

## P0 — Critical (блокирует коммерческую готовность)

### TRUST-002: Integrations mock toggle
- **Страница**: /app/integrations
- **Проблема**: Toggle "Connect" показывает success notification, но реального подключения не происходит
- **Влияние**: Пользователь думает что интеграция настроена, но это не так. При попытке использовать — ничего не работает.
- **Fix**: Либо реальная интеграция (OAuth для SaaS, API для REST), либо явная пометка "Демо" на toggle

### TRUST-003: Scanner fake progress bar
- **Страница**: /app/scanner
- **Проблема**: Progress bar сканирования симулируется, не отражает реальный прогресс backend
- **Влияние**: Пользователь не доверяет результатам сканирования
- **Fix**: Подключить SSE от backend для real-time progress, либо убрать progress bar

### TRUST-004: Email reporting mock
- **Страница**: /app/reports/email
- **Проблема**: Email sending показывает success, но реальная отправка не происходит
- **Влияние**: Пользователь думает что отчёт отправлен, но получатель его не получает
- **Fix**: Реальный SMTP (Resend/SendGrid) или явная пометка "Симуляция"

## P1 — High (снижает доверие)

### TRUST-001: Demo data без пометки
- **Страница**: /app/dashboard (legacy)
- **Проблема**: Demo данные отображаются как реальные метрики без явной пометки DemoBadge на всех графиках
- **Влияние**: Пользователь принимает mock за реальные данные
- **Fix**: DemoBadge на всех секциях с demo данными (частично сделано в INT-038)

### TRUST-005: Notification rules localStorage
- **Страница**: /app/notifications
- **Проблема**: Rules CRUD сохраняется в localStorage, но UI не показывает что данные не персистентны
- **Влияние**: При смене браузера данные теряются
- **Fix**: Пометка "Локально" или backend persistence

### TRUST-006: API keys mock
- **Страница**: /app/api-keys
- **Проблема**: API keys создаются локально, не валидируются
- **Влияние**: Пользователь думает что keys работают, но API их не проверяет
- **Fix**: Backend должен валидировать API keys

---

## P2 — Medium (улучшить)

### TRUST-007: Marketplace install client-side
- **Страница**: /app/marketplace
- **Проблема**: Install/uninstall работает client-side, реальная установка не происходит
- **Влияние**: Пользователь думает что плагин установлен, но он не работает
- **Fix**: DemoBadge на всех карточках (частично сделано)

### TRUST-008: Workspace mock activity
- **Страница**: /app/workspace/*
- **Проблема**: 5 страниц с hardcoded activity/stats
- **Влияние**: Выглядит ненастояще
- **Fix**: Реальные данные или empty states

### TRUST-009: Magic numbers
- **Страницы**: /app/command-center, /app/dashboard
- **Проблема**: Числа вроде "94%", "82%" без объяснения откуда взялись
- **Влияние**: Пользователь не понимает что значат эти проценты
- **Fix**: Tooltip с объяснением или убрать из User mode

### TRUST-010: Placeholder text
- **Страницы**: различные
- **Проблема**: Lorem ipsum-like placeholder text в некоторых местах
- **Влияние**: Выглядит незаконченно
- **Fix**: Реальный copy или empty state
