# Пути атак (Attack Paths) SIP

## Обзор

Модуль Attack Paths превращает разрозненный список уязвимостей в осмысленные сценарии атак, связывая отдельные слабости в цепочки, которые показывают реалистичный путь злоумышленника от первоначального проникновения до компрометации критичных активов. Модуль базируется на фреймворке MITRE ATT&CK — международном стандарте описания тактик, техник и процедур (TTP) кибератак. Каждая обнаруженная уязвимость автоматически маппится на соответствующую технику ATT&CK, после чего алгоритм строит граф возможных атак с оценкой вероятности и ущерба для каждого маршрута.

---

## Связь с MITRE ATT&CK

MITRE ATT&CK — это структурированная модель поведения злоумышленников, описывающая 14 тактик (tactics) на уровне Enterprise, от Reconnaissance (Разведка) до Exfiltration (Экфильтрация) и Impact (Воздействие). Каждая тактика включает множество техник (techniques), описывающих конкретные способы достижения цели атакующего. SIP использует ATT&CK как словарь для классификации обнаруженных уязвимостей и как основу для моделирования цепочек атак.

### Маппинг уязвимостей на техники ATT&CK

Каждая находка в SIP автоматически маппится на одну или несколько техник ATT&CK на основе:

- **Типа уязвимости** — SQL-инъекция → T1190 (Exploit Public-Facing Application), слабые учётные данные → T1078 (Valid Accounts).
- **Затронутого сервиса** — RDP → T1021.001 (Remote Services: Remote Desktop Protocol), SSH → T1021.004 (Remote Services: SSH).
- **Вектора эксплуатации** — фишинг → T1566 (Phishing), supply chain → T1195 (Supply Chain Compromise).
- **Влияния** — утечка данных → T1567 (Exfiltration Over Web Service), шифрование файлов → T1486 (Data Encrypted for Impact).

Маппинг осуществляется через встроенную базу соответствий, которая регулярно обновляется при выходе новых версий ATT&CK.

## Моделирование путей атак

### Построение графа атак

Алгоритм строит ориентированный граф, в котором узлы — это техники ATT&CK и целевые активы, а рёбра — переходы между техниками, основанные на реальных сценариях атак. Граф строится по следующим правилам:

1. **Точка входа (Entry Point)** — узлы, соответствующие тактике Initial Access (Tactic TA0001). Это публично доступные сервисы с уязвимостями, фишинговые векторы, скомпрометированные учётные данные.
2. **Промежуточные шаги** — техники Execution, Persistence, Privilege Escalation, Defense Evasion, Credential Access, Lateral Movement, каждая из которых может быть реализована через обнаруженную уязвимость или неправильную конфигурацию.
3. **Целевой актив (Objective)** — критичный для бизнеса актив (база данных с персональными данными, контроллер домена, финансовая система), достижение которого означает серьёзный инцидент.

### Оценка риска пути

Каждый путь атак получает совокупную оценку риска, рассчитываемую по формуле:

```
Risk(path) = Π(probability_i) × Impact(objective)
```

Где `probability_i` — вероятность успешной эксплуатации i-го шага (определяется CVSS-оценкой, наличием публичного эксплойта, сложностью эксплуатации), а `Impact(objective)` — оценка ущерба при компрометации целевого актива (определяется классификацией актива и бизнес-контекстом).

Пути сортируются по убыванию совокупного риска, что позволяет аналитику фокусироваться на наиболее вероятных и разрушительных сценариях.

## Визуализация

Пути атак отображаются в интерактивном интерфейсе в виде линейной диаграммы (kill chain) и в виде графа. Каждый шаг содержит:

- Иконку тактики ATT&CK (Initial Access → Execution → Persistence → ... → Impact).
- Название техники и её ID (например, T1190 — Exploit Public-Facing Application).
- Затронутый актив и конкретную уязвимость.
- CVSS-оценку и наличие публичного эксплойта.
- Статус контрмеры (если применимо).

Аналитик может развернуть любой шаг для получения детальной информации, включая proof-of-concept уязвимости и рекомендацию по устранению. Переключение между путями осуществляется через боковую навигацию с мини-превью каждого маршрута.

---

## Приоритизация защиты

На основе анализа путей атак модуль формирует приоритизированный список мер по защите. Принцип приоритизации: **устранение одного узла в графе атак может заблокировать множество путей**. Например, закрытие порта RDP на пограничном сервере может разорвать 15 из 23 моделируемых путей атак. AI-рекомендации интегрируются с анализом путей, предлагая контрмеры с наибольшим эффектом при наименьших затратах, что позволяет оптимально распределять ограниченные ресурсы команды безопасности.

## Справочник MITRE ATT&CK

Релевантные тактики, используемые в моделировании путей атак SIP:

| Тактика              | ID       | Описание                                         |
|----------------------|----------|---------------------------------------------------|
| Initial Access       | TA0001   | Способы первоначального проникновения в сеть      |
| Execution            | TA0002   | Выполнение вредоносного кода на скомпрометированной системе |
| Persistence          | TA0003   | Сохранение доступа к скомпрометированной системе  |
| Privilege Escalation | TA0004   | Повышение привилегий на скомпрометированной системе |
| Defense Evasion      | TA0005   | Обход средств защиты и мониторинга                |
| Credential Access    | TA0006   | Получение учётных данных                          |
| Lateral Movement     | TA0008   | Перемещение по сети от одной системы к другой     |
| Exfiltration         | TA0010   | Несанкционированный вывод данных из сети          |
| Impact               | TA0040   | Нарушение доступности, целостности или конфиденциальности |
