# PRODUCT_VISION.md — Продуктовое видение

---

## Что мы строим

SIP — это не сканер уязвимостей.

SIP — это операционная система для безопасности бизнеса.

Разница фундаментальна. Сканер находит проблемы. Операционная система помогает их понять, приоритизировать и устранить — так, чтобы бизнес принимал обоснованные решения.

---

## Что мы продаём

### Чувство контроля

Владелец бизнеса открывает платформу и видит понятную картину: что защищено, где есть риски, что нужно сделать прямо сейчас. Не список из 200 CVE — а 3 конкретных действия, которые устранят 80% риска.

### Понятность

Каждый элемент интерфейса объясняет себя. Не нужно знать, что такое CVE-2024-1234 — платформа скажет: «На вашем сервере открыт Redis без пароля. Это значит, что любой может прочитать ваши данные. Исправить за 5 минут».

### Уверенность

Пользователь закрывает платформу с ощущением: «Я знаю, что происходит. Я знаю, что делать. Я контролирую ситуацию». Не страх от красных алертов — а спокойствие от понимания.

---

## Что мы НЕ продаём

- Ещё один сканер уязвимостей
- Список CVE для технических специалистов
- Инструмент, требующий обучения
- Дашборд с красивыми графиками, за которыми нет действий
- Платформу, которая показывает проблемы без решений

---

## Для кого

### Первичная аудитория

**Владелец / руководитель бизнеса** (CEO, CTO, CISO), который:
- Понимает, что безопасность важна
- Не хочет разбираться в технических деталях
- Хочет понимать: «Насколько мы защищены?» и «Что делать?»
- Принимает решения на основе рисков, а не технических отчётов

### Вторичная аудитория

**Инженер / DevOps**, который:
- Выполняет рекомендации руководства
- Нуждается в конкретных шагах для устранения проблем
- Использует технические данные (CVE, конфигурации, логи)
- Настраивает автоматизацию и интеграции

### Третичная аудитория

**Аналитик безопасности**, который:
- Анализирует тренды рисков
- Формирует отчёты для соответствия требованиям
- Работает с Knowledge Graph и Attack Paths

## Ключевые отличия

| Традиционный подход | SIP |
|---------------------|-----|
| Сканер находит 200 проблем | Платформа показывает 3 приоритетных |
| «Critical: CVE-2024-1234» | «Открытый Redis без пароля — любой может прочитать данные» |
| Технический отчёт на 50 страниц | Executive Summary: 3 пункта, понятные бизнесу |
| Нужно знать, что делать | Платформа говорит, что делать и почему |
| Красные алерты вызывают панику | «Есть возможность улучшить» — вызывает доверие |
| Один размер для всех | Адаптируется под роль: CEO, DevOps, инженер |
| Результаты в вакууме | Связи: уязвимость → путь атаки → бизнес-риск |
| Нужно учиться пользоваться | Платформа объясняет сама себя |

## Продуктовые ценности

### 1. Понятность важнее полноты

Лучше показать 3 ключевых риска с объяснением, чем 200 без контекста.

### 2. Действие важнее информации

Каждый элемент интерфейса должен вести к действию. Если пользователь видит проблему — он должен видеть и решение.

### 3. Доверие важнее функциональности

Красивый дашборд, за которым нет реальных данных, разрушает доверие. Простой интерфейс с честной информацией — укрепляет.

### 4. Адаптация важнее стандартизации

CEO и DevOps видят одну и ту же проблему по-разному. И это правильно. Платформа подстраивается, а не заставляет подстраиваться.

### 5. Прогресс важнее перфекционизма

«Устранено 3 из 10 рисков» — это успех. «0 из 10» потому что интерфейс пугает — это провал.
