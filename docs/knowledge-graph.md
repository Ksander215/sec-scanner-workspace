# Граф знаний (Knowledge Graph) SIP

## Обзор

Граф знаний — аналитический модуль SIP, визуализирующий взаимосвязи между активами инфраструктуры, обнаруженными уязвимостями, техниками атак и мерами защиты. В отличие от традиционных списков уязвимостей, граф знаний раскрывает контекст: как одна уязвимость может привести к компрометации другого актива, какие техники MITRE ATT&CK применимы к конкретной конфигурации, и какие активы оказываются наиболее критичными в контексте цепочек атак. Модуль автоматически строит и обновляет граф при каждом новом сканировании, обеспечивая актуальность аналитики без ручного вмешательства.

---

## Типы узлов

Граф знаний оперирует следующими типами узлов, каждый из которых имеет уникальный набор атрибутов:

- **Asset (Актив)** — хост, сервер, контейнер, виртуальная машина, сетевой сервис. Атрибуты: IP-адрес, hostname, ОС, открытые порты, запущенные сервисы, принадлежность к проекту.
- **Vulnerability (Уязвимость)** — обнаруженная слабость в безопасности. Атрибуты: CVE-идентификатор, CVSS-оценка, тип, статус, дата обнаружения, маппинг на MITRE ATT&CK.
- **Technique (Техника)** — техника атаки из фреймворка MITRE ATT&CK. Атрибуты: ID техники (например, T1190), название, тактика, описание, источники информации.
- **Mitigation (Контрмера)** — мера по устранению или снижению риска. Атрибуты: тип (патч, конфигурация, WAF-правило), статус внедрения, эффективность.

## Типы связей

Рёбра графа представляют различные типы отношений между узлами:

- **HAS_VULNERABILITY** — актив содержит уязвимость. Вес ребра — CVSS-оценка.
- **EXPLOITED_BY** — уязвимость может быть эксплуатирована с использованием указанной техники MITRE ATT&CK. Вес — вероятность эксплуатации.
- **CONNECTED_TO** — сетевая связность между активами. Вес — тип соединения (TCP, HTTP, SSH, база данных).
- **DEPENDS_ON** — зависимость одного актива от другого (например, веб-сервер зависит от базы данных). Критично для расчёта каскадного риска.
- **MITIGATED_BY** — уязвимость снижена контрмерой. Вес — остаточный риск.

## Визуализация и взаимодействие

Граф знаний отображается в интерактивном интерфейсе на базе D3.js с WebGL-рендерингом для графов размером более 5000 узлов. Поддерживаемые взаимодействия:

- **Масштабирование и панорамирование** — свободная навигация по графу.
- **Фильтрация** — отображение узлов определённого типа, критичности, статуса. Скрытие исправленных уязвимостей для фокуса на активных проблемах.
- **Трассировка пути** — выделение пути между двумя выбранными узлами с подсветкой промежуточных узлов и рёбер. Позволяет увидеть, как атакующий может пройти от точки входа до критичного актива.
- **Детализация узла** — клик по узлу открывает боковую панель с полной информацией, включая историю изменений и связанные находки.
- **Экспорт** — сохранение текущего представления в SVG или PNG для включения в отчёты и презентации.

## Алгоритмы анализа

Модуль реализует несколько графовых алгоритмов для автоматической аналитики:

- **PageRank** — ранжирование активов по их значимости в графе. Активы с высоким PageRank являются критичными узлами, компрометация которых оказывает наибольшее влияние на инфраструктуру.
- **Betweenness Centrality** — выявление узлов-«мостов», через которые проходит наибольшее количество кратчайших путей. Такие узлы являются приоритетными целями для защиты.
- **Shortest Path** — поиск кратчайшего пути атаки между двумя узлами. Используется модулем Attack Paths для моделирования сценариев.
- **Community Detection** — автоматическая кластеризация активов в группы по сетевой близости и типу уязвимостей. Помогает выявить системные проблемы в конкретных сегментах инфраструктуры.

## API и интеграция

Граф знаний доступен через REST API (`GET /projects/:id/knowledge-graph`) с параметрами фильтрации и ограничения глубины. Результат возвращается в формате узлов и рёбер, совместимом с D3.js и Cytoscape. Для программной аналитики доступен GraphQL-эндпоинт, позволяющий выполнять произвольные запросы к графу, включая агрегации и путь-запросы.
