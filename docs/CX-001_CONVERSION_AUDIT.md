## BLOCK 2 — Анализ каждого экрана

### Экран 1: Hero (/app/home)

**Что пользователь хочет сделать?**
Понять, что это за продукт и зачем он ему.

**Что его может остановить?**
- Кнопка "Проверить" disabled, пока не введён домен — но нет подсказки, какой формат вводить
- "AI Assistant" секция ниже — пользователь может не дойти, если не введёт домен
- Sidebar перегружен (16 пунктов в User mode) — может отвлечь

**Что вызывает недоверие?**
- "Статус: Хорошо" — но сканирование не запускалось. Откуда "Хорошо"?
- Результат проверки показывает 3 проблемы для example.com, но это demo данные — пользователь может подумать, что это реальные проблемы его сайта

**Что вызывает эмоцию?**
- ✅ "Введите адрес вашего сайта — AI проверит за 2 минуты" — вызывает любопытство
- ✅ Trust badges (AI переводит, честные результаты, 2 минуты) — снимают тревогу
- ❌ Demo результат без пометки "Demo" — может вызвать раздражение, когда пользователь поймёт

**Что мотивирует двигаться дальше?**
- Кнопка "Проверить" — чёткий CTA
- AI prompts ("Проверь мой сайт", "Что исправить первым?") — предлагают следующий шаг

### Экран 2: Сканирование (/app/scanner)

**Что пользователь хочет сделать?**
Запустить проверку своего сайта.

**Что его может остановить?**
- Сложная форма: Project → Source → Tools → Run — слишком много шагов
- Выбор инструментов (nmap, nuclei, ZAP) — технические термины
- Pipeline stages — непонятно пользователю

**Что вызывает недоверие?**
- Progress bar симулируется (TRUST-003) — пользователь видит "проверка идёт", но реального сканирования нет

**Что вызывает эмоцию?**
- ✅ Кнопка "Начать сканирование" — мотивирует
- ❌ Технические детали инструментов — пугают нетехнического пользователя

**Что мотивирует двигаться дальше?**
- Результат сканирования → переход к Findings/Security Review

---

### Экран 3: Отчёт (/app/security-review)

**Что пользователь хочет сделать?**
Понять результаты и что делать.

**Что его может остановить?**
- 8 проблем — может показаться слишком много
- "112 часов" на исправление — может испугать

**Что вызывает недоверие?**
- Demo данные (example.com) без пометки "Demo"
- Нет реальной связи с тем, что пользователь ввёл на Home

**Что вызывает эмоцию?**
- ✅ AI Executive Summary — "Проверка выявила 3 критическую проблему" — чётко и понятно
- ✅ "Что исправить первым" — топ 5 действий, отсортированных по бизнес-эффекту
- ✅ "Скопировать задачу разработчику" — практичная кнопка
- ✅ "Объяснить проще" — снижает тревогу
- ❌ "112 часов" — может отпугнуть

**Что мотивирует двигаться дальше?**
- "Получите подробный план исправления" → Pricing — естественный переход

---

### Экран 4: Pricing (/app/pricing)

**Что пользователь хочет сделать?**
Понять сколько стоит и выбрать план.

**Что его может остановить?**
- Цены в рублях (₽/мес) — но pricing.ts определяет в долларах. Расхождение!
- Только 3 плана видны (Бесплатно, Профессионал, Бизнес) — где Enterprise?
- Нет "14-day free trial" для Pro/Business — только "Начать бесплатно" для Free
- Нет comparison table — непонятно что получаешь за деньги

**Что вызывает недоверие?**
- Цены в рублях — может показаться локальным продуктом, не международным
- "Бесплатно" план — но что в нём входит? Нет деталей
- Нет testimonials / social proof

**Что вызывает эмоцию?**
- ✅ "Популярный" бейдж на Профессионал — социальное доказательство
- ❌ Нет "14-day free trial" — упущенная возможность

**Что мотивирует двигаться дальше?**
- "Попробовать бесплатно" — но ведёт ли это к регистрации? Нет auth.

## BLOCK 5 — Топ-5 улучшений для немедленной реализации

### 1. Demo пометка на результатах (Home + Security Review)
**Проблема**: Пользователь видит 3 критических проблемы для "example.com" и думает что это его проблемы
**Решение**: Добавить "Demo" бейдж к результатам
**Влияние**: Критическое — без этого пользователь теряет доверие

### 2. "14-day free trial" на Pricing
**Проблема**: Нет trial CTA для Pro/Business — пользователь не видит пути к пробе
**Решение**: Добавить "Try Pro free for 14 days" кнопку
**Влияние**: Высокое — напрямую влияет на trial signups

### 3. Enterprise план на Pricing
**Проблема**: Только 3 плана видны, Enterprise ($4999) отсутствует
**Решение**: Добавить 4-й тариф из pricing.ts
**Влияние**: Высокое — открывает enterprise segment

### 4. Убрать "Статус: Хорошо" с Home
**Проблема**: "Хорошо" без сканирования — подрывает доверие
**Решение**: Заменить на "Давайте проверим вашу безопасность"
**Влияние**: Критическое — честность > впечатления

### 5. "Это пример" пометка на demo результате Home
**Проблема**: Результат для example.com выглядит как реальный
**Решение**: Добавить "Это пример. Введите свой домен для реальной проверки"
**Влияние**: Высокое — предотвращает недопонимание
