# Маркетплейс SIP

## Обзор

Маркетплейс SIP — это централизованный каталог расширений, интеграций и шаблонов, расширяющих возможности платформы. Маркетплейс объединяет официальные плагины, разработанные командой SIP, и сторонние расширения от сообщества. Каждое расширение проходит автоматическую проверку безопасности и совместимости перед публикацией, что гарантирует стабильность работы платформы. Маркетплейс доступен через веб-интерфейс платформы и через CLI, что позволяет автоматизировать управление расширениями в CI/CD-пайплайнах.

## Категории расширений

### Интеграции (Integrations)

Коннекторы к популярным корпоративным системам для автоматизации рабочих процессов безопасности:

- **Jira** — автоматическое создание тикетов на устранение уязвимостей с привязкой к находке в SIP. Поддержка кастомных полей, автоматический переход статусов при изменении в SIP.
- **Slack** — уведомления о критических находках в заданных каналах, дайджесты за период, интерактивные кнопки для быстрого действия (подтвердить, пометить false positive, назначить ответственного).
- **Microsoft Teams** — аналогично Slack, с адаптивными карточками Teams.
- **ServiceNow** — двусторонняя синхронизация инцидентов безопасности между SIP и ServiceNow ITSM.
- **PagerDuty** — эскалация критических инцидентов по расписанию дежурных.
- **Telegram** — бот для мгновенных уведомлений и управления сканированиями.

### Дашборды (Dashboards)

Готовые наборы виджетов для конкретных ролей и задач:

- **Security Overview** — общая панель безопасности с метриками, трендами и hotspot-картой.
- **Compliance Tracker** — мониторинг соответствия стандартам (ISO 27001, SOC 2, PCI DSS) с прогресс-барами по каждому контролу.
- **Vulnerability Trends** — детальная аналитика трендов уязвимостей за произвольный период с группировкой по источнику, типу и критичности.
- **Asset Inventory** — инвентаризация активов с данными об открытых портах, сервисах и связанных уязвимостях.

### Шаблоны отчётов (Report Templates)

Профессиональные шаблоны для генерации отчётов в различных форматах:

- **Executive Summary** — краткий отчёт для руководства с ключевыми метриками и рекомендациями.
- **SOC 2 Compliance** — отчёт по соответствию критериям SOC 2 Type II.
- **ISO 27001 Audit** — аудиторский отчёт по ISO/IEC 27001:2022.
- **PCI DSS Assessment** — оценка соответствия PCI DSS v4.0.
- **Penetration Test Report** — шаблон отчёта о пенетестировании с доказательствами и рекомендациями.

### AI-промпты (AI Prompts)

Коллекция оптимизированных промптов для AI-рекомендаций, адаптированных под конкретные сценарии:

- **Remediation Prioritizer** — промпт для приоритизации мер по устранению с учётом бизнес-контекста.
- **Risk Communicator** — промпт для формирования понятных объяснений рисков для не-технической аудитории.
- **Compliance Mapper** — промпт для маппинга обнаруженных уязвимостей на требования конкретного стандарта.

### Темы оформления (Themes)

Альтивные визуальные темы для интерфейса SIP:

- **Dark Pro** — профессиональная тёмная тема с высокой контрастностью.
- **Light Corporate** — светлая тема в корпоративном стиле.
- **High Contrast** — тема с максимальной контрастностью для пользователей с нарушениями зрения.

## Установка и управление

### Через веб-интерфейс

Откройте раздел **Marketplace** в навигации платформы, найдите нужное расширение через поиск или каталог категорий, нажмите «Установить» и подтвердите запрашиваемые права доступа.

### Через CLI

```bash
# Поиск плагинов
sip marketplace search "kubernetes"

# Установка плагина
sip plugin install sip-k8s-scanner

# Обновление
sip plugin update sip-k8s-scanner

# Удаление
sip plugin uninstall sip-k8s-scanner

# Список установленных
sip plugin list
```

### Управление версиями

Маркетплейс отслеживает версии каждого плагина и уведомляет о доступных обновлениях. Обновления проверяются на совместимость с текущей версией SIP. Критические обновления безопасности помечаются соответствующим бейджем и рекомендуются к немедленной установке. Откат на предыдущую версию поддерживается через CLI-команду `sip plugin rollback`.

## Публикация расширений

Для публикации расширения в Маркетплейсе необходимо:

1. Зарегистрировать аккаунт разработчика на portal.sip-platform.dev.
2. Подготовить расширение в соответствии с требованиями (валидный манифест, тесты, документация).
3. Пройти автоматическую проверку: линтинг кода, сканирование безопасности, тесты совместимости.
4. Пройти ручное ревью от команды SIP (для первых публикаций).
5. Опубликовать через CLI: `sip plugin publish`.

Подробная инструкция по разработке расширений доступна в разделе [Документация плагинов](./plugins.md).
