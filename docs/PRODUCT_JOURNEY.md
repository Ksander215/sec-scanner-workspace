# PRODUCT_JOURNEY.md — Путь пользователя

> EP-001 BLOCK 9. Полный путь от Landing до оплаты подписки.

---

## Карта путей

```
Landing → Регистрация → First Scan → First Report → AI рекомендации →
Исправление → Повторное сканирование → История → Подписка → Enterprise
```

---

## Этап 1: Landing

**Цель**: посетитель понимает ценность за 30 секунд.

**Текущее состояние**: ✅ Landing page существует, рассказывает о платформе.

**Что проверить**:
- Hero section: "Понимай безопасность без инженера" — понятно?
- Demo preview: показывает реальный продукт?
- Pricing: понятны планы?
- CTA: "Попробовать бесплатно" — куда ведёт?

**Пробел**: Нет регистрации. CTA "Попробовать" ведёт на /app без auth.

**Приоритет**: P1 — нужна реальная регистрация (EP-002).

---

## Этап 2: Регистрация

**Цель**: пользователь создаёт аккаунт за < 2 минут.

**Текущее состояние**: ❌ Нет системы аутентификации (PLAT-008 SSO not_started).

**Что нужно**:
- Email/password регистрация
- Google OAuth (опционально)
- Email verification
- Welcome email с onboarding

**Пробел**: Полностью отсутствует.

**Приоритет**: P0 — критично для коммерции.

---

## Этап 3: First Scan

**Цель**: пользователь запускает первое сканирование за < 5 минут после регистрации.

**Текущее состояние**: ✅ Scanner работает (клиент-side), но прогресс симулируется (TRUST-003).

**Что проверить**:
- Onboarding wizard: есть? (нет)
- AI Assistant first-visit: ✅ есть
- Простой flow: выбрать цель → нажать "Проверить" → увидеть результат

**Пробел**:
- Нет onboarding wizard
- Progress bar fake (TRUST-003)
- Нет реального backend сканирования (backend на INT-036)

**Приоритет**: P0 — закрыть TRUST-003, P1 — onboarding wizard.

---

## Этап 4: First Report

**Цель**: пользователь получает понятный отчёт.

**Текущее состояние**: ✅ Reports работают (HTML/JSON/SARIF/MD/CSV).

**Что проверить**:
- Executive Summary: на языке бизнеса?
- Visual: понятные графики?
- Download: работает?

**Пробел**: Форматы SARIF/JSON технические — скрыть за "Дополнительные форматы".

**Приоритет**: P2 — упростить форматы.

---

## Этап 5: AI рекомендации

**Цель**: AI предлагает что исправить первым.

**Текущее состояние**: ✅ AIS Assistant работает, даёт рекомендации.

**Что проверить**:
- AI Assistant: ✅ visible, открывается
- Recommendations: ✅ на Command Center
- Context: ✅ page-aware

**Пробел**: AI называется "Adaptive Intelligence System" — переименовать в "AI Assistant".

**Приоритет**: P1 — переименование (EP-001 BLOCK 7).

---

## Этап 6: Исправление

**Цель**: пользователь исправляет уязвимости по рекомендациям AI.

**Текущее состояние**: ⚠️ AI даёт рекомендации, но нет tracking исправлений.

**Что нужно**:
- "Отметить как исправлено" кнопка на findings
- Progress tracking по goal
- AI подтверждает исправление

**Пробел**: Нет tracking исправлений.

**Приоритет**: P2 — добавить tracking.

---

## Этап 7: Повторное сканирование

**Цель**: пользователь проверяет что исправления помогли.

**Текущее состояние**: ✅ Scanner может запуститься повторно.

**Пробел**: Нет сравнения "до/после".

**Приоритет**: P2 — добавить diff view.

---

## Этап 8: История

**Цель**: пользователь видит прогресс во времени.

**Текущее состояние**: ✅ Scan history в localStorage.

**Пробел**: Нет server-side persistence, нет трендов.

**Приоритет**: P1 — backend persistence (EP-002).

---

## Этап 9: Подписка

**Цель**: пользователь оплачивает Pro план.

**Текущее состояние**: ❌ Нет billing системы.

**Что нужно**:
- Pricing page: ✅ есть
- Checkout: ❌ нет
- Stripe integration: ❌ нет
- Trial period: ❌ нет

**Пробел**: Полностью отсутствует billing.

**Приоритет**: P0 для коммерциализации (EP-003).

---

## Этап 10: Enterprise

**Цель**: компания оплачивает Enterprise план ($2499/мес).

**Текущее состояние**: ❌ Нет enterprise features (SSO, RBAC, audit trail).

**Что нужно**:
- SSO (SAML, Google Workspace)
- RBAC (роли, разрешения)
- Audit trail (логи действий)
- On-premise опция

**Пробел**: PLAT-008 (SSO), PLAT-010 (RBAC), PLAT-012 (Audit) — все not_started.

**Приоритет**: P1 для enterprise sales (EP-004+).

## Критический путь к коммерции

1. **Регистрация** (P0) — без пользователей нет бизнеса
2. **Подписка** (P0) — без оплаты нет revenue
3. **First Scan** (P0) — без value нет retention
4. **AI рекомендации** (P1) — ключевое отличие от конкурентов
5. **История** (P1) — накопительная ценность
