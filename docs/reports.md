# Отчёты SIP

## Обзор

Модуль отчётов SIP позволяет генерировать профессиональные документы на основе результатов сканирования и аналитики. Отчёты адаптированы под различные аудитории: от технических специалистов, которым нужна полная детализация каждой находки, до руководителей и совета директоров, ожидающих агрегированную бизнес-аналитику. Модуль поддерживает шесть форматов экспорта, шаблоны для соответствия нормативным стандартам и гибкую настройку содержимого через параметры генерации. Каждый отчёт включает визуальные элементы — графики, диаграммы, карты рисков — и может быть локализован на русский или английский язык.

---

## Форматы экспорта

### PDF

Наиболее востребованный формат для предоставления отчётов руководству, аудиторам и заказчикам. PDF-отчёт генерируется с помощью ReportLab и включает:

- Титульную страницу с логотипом организации, названием проекта и датой.
- Оглавление с кликабельными ссылками на разделы.
- Резюме рисков — одностраничная сводка с ключевыми метриками, индексом безопасности и динамикой по сравнению с предыдущим отчётом.
- Детализированный список уязвимостей с CVSS-оценками, описаниями и рекомендациями.
- Визуализации: круговые диаграммы по критичности, графики трендов, heatmap-карта активов.
- Приложение с маппингом на MITRE ATT&CK и путями атак.

PDF поддерживает векторную графику, внедрение шрифтов (включая кириллические) и совместим с PDF/A для длительного архивирования.

### DOCX

Формат Microsoft Word для организаций, которым необходимо редактировать отчёт перед финальным предоставлением. DOCX-отчёт сохраняет структуру, форматирование и визуализации PDF, но позволяет аналитику добавить комментарии, скорректировать формулировки и дополнить разделы. Генерация осуществляется через библиотеку docx.js с кастомными стилями, соответствующими брендбуку SIP.

### XLSX

Табличный формат для дальнейшей обработки в Excel, Google Sheets или BI-системах. Каждая находка представлена строкой с полями: идентификатор, актив, порт, тип уязвимости, CVSS, статус, ответственный, дата обнаружения, маппинг MITRE ATT&CK. Листы: «Все находки», «По критичности», «По активам», «Тренды», «Статистика». Условное форматирование подсвечивает критические строки красным, высокие — оранжевым.

### JSON

Машинно-читаемый формат для интеграции с другими системами и автоматизации обработки. JSON-отчёт содержит полную структурированную информацию: метаданные проекта, массив находок с вложенными объектами (актив, CVSS-вектор, маппинг ATT&CK, рекомендации), агрегированную статистику и пути атак. Формат соответствует OpenAPI-схеме SIP и может быть валидирован автоматически.

### CSV

Упрощённый табличный формат для импорта в SIEM-системы, базы данных и аналитические платформы. CSV содержит плоскую таблицу находок с основными полями. Кодировка UTF-8 с BOM для корректного отображения кириллицы в Excel.

### HTML

Интерактивный веб-отчёт, открываемый в любом браузере. HTML-отчёт включает интерактивные графики (Recharts), раскрывающиеся секции, фильтрацию по критичности и статусу, а также возможность навигации по разделам. Подходит для публикации на внутреннем портале компании или предоставления по ссылке.

## Шаблоны отчётов

SIP поставляется с набором преднастроенных шаблонов:

| Шаблон              | Аудитория        | Содержание                                                    |
|----------------------|-------------------|---------------------------------------------------------------|
| Executive Summary    | C-level, Совет    | Резюме рисков, ROI безопасности, тренды, ключевые рекомендации |
| Technical Report     | Security Team     | Полный перечень находок, PoC, маппинг MITRE ATT&CK            |
| Compliance (ISO 27001)| Аудиторы         | Соответствие контролям, пробелы, план закрытия               |
| Compliance (SOC 2)  | Аудиторы          | Оценка по критериям TSC, доказательства                      |
| Compliance (PCI DSS)| QSA, Аудиторы     | Требования PCI DSS v4.0, статус соответствия                  |
| Penetration Test     | Заказчик пенетеста| Методология, находки, exploitation, рекомендации              |

Кастомные шаблоны можно создать через систему плагинов (тип `report`).

## Параметры генерации

При создании отчёта пользователь указывает:

- **Проект** — исходные данные для отчёта.
- **Формат** — один из шести поддерживаемых.
- **Шаблон** — структура и содержимое.
- **Период** — диапазон дат для включения находок.
- **Фильтры** — по критичности, статусу, типу уязвимости, активу.
- **Язык** — русский или английский.
- **Включения** — графики, пути атак, рекомендации, историческая аналитика.

Генерация отчёта — асинхронный процесс. Статус отслеживается через API (`GET /reports/:id`), готовый отчёт скачивается через `GET /reports/:id/download`.
