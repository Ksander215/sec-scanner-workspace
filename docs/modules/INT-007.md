# INT-007 — Recommendation Engine

## Overview

The Recommendation Engine generates actionable remediation guidance for security findings, prioritised by risk and feasibility. It produces `Recommendation` objects containing specific `RemediationAction` steps, then aggregates them into a `RemediationPlan` organised by `RemediationPhase` (immediate, short-term, long-term). The engine balances urgency, effort, and dependency ordering to produce a practical, sequenced plan.

Key responsibilities:

- **Recommendation generation** — Produce concrete remediation actions for each finding based on its category, severity, and risk score.
- **Priority assignment** — Classify recommendations by `RecommendationPriority` (critical, high, medium, low).
- **Plan creation** — Aggregate recommendations into a phased `RemediationPlan` with dependency-aware ordering.
- **Status tracking** — Each recommendation has a `RecommendationStatus` for lifecycle management.

The action generator uses a catalogue of category-specific remediation templates. The dependency resolver ensures that prerequisite actions are ordered before dependent ones. The phase sorter groups actions into immediate (0–24 h), short-term (1–30 d), and long-term (30+ d) windows.

## Data Flow

```
1.  generate(findings, risks):
    a.  For each finding:
        i.   Resolve its category → select appropriate action templates.
        ii.  Generate RemediationActions with specific steps.
        iii. Assign priority based on risk score and category.
    b.  Return Recommendation[] with actions and priorities.

2.  createPlan(recommendations):
    a.  Resolve dependencies between actions (e.g. "patch X before restarting Y").
    b.  Sort actions into RemediationPhase buckets.
    c.  Order actions within each phase by priority and dependency.
    d.  Return RemediationPlan.
```

## Public API

### Class: `RecommendationEngine`

| Method | Signature | Description |
|--------|-----------|-------------|
| `generate` | `generate(findings: SecurityFinding[], risks: RiskAssessment[]): Recommendation[]` | Generate recommendations for each finding based on its category and risk. |
| `createPlan` | `createPlan(recommendations: Recommendation[]): RemediationPlan` | Aggregate recommendations into a phased, dependency-ordered plan. |

### Types

#### `Recommendation`

```typescript
interface Recommendation {
  id: string;
  findingId: string;
  title: string;
  description: string;
  priority: RecommendationPriority;
  status: RecommendationStatus;
  actions: RemediationAction[];
  estimatedEffort: string;      // e.g. "2 hours", "1-2 days"
  dependencies: string[];       // IDs of other recommendations that must be done first
  references: string[];         // links to docs, CVEs, etc.
}
```

#### `RemediationAction`

```typescript
interface RemediationAction {
  id: string;
  type: "patch" | "configure" | "replace" | "remove" | "monitor" | "investigate";
  title: string;
  description: string;
  command?: string;              // shell command or code snippet
  automated: boolean;            // can this be run automatically?
  verification: string;          // how to verify the action succeeded
}
```

#### `RemediationPlan`

```typescript
interface RemediationPlan {
  id: string;
  phases: RemediationPhase[];
  totalActions: number;
  estimatedTotalEffort: string;
  criticalPathLength: number;    // longest dependency chain
}
```

#### `RemediationPhase`

```typescript
interface RemediationPhase {
  name: string;                  // "immediate" | "short_term" | "long_term"
  timeframe: string;             // "0-24 hours" | "1-30 days" | "30+ days"
  recommendations: Recommendation[];
  actionCount: number;
}
```

#### `RecommendationPriority`

```typescript
enum RecommendationPriority {
  Critical = "critical",
  High = "high",
  Medium = "medium",
  Low = "low",
}
```

#### `RecommendationStatus`

```typescript
enum RecommendationStatus {
  Pending = "pending",
  InProgress = "in_progress",
  Completed = "completed",
  Accepted = "accepted",       // risk accepted, no action needed
  Deferred = "deferred",
  Dismissed = "dismissed",
}
```

## Extension Points

1. **Custom action templates** — The action generator uses a catalogue of templates per `FindingCategory`. Register new templates or override existing ones for organisation-specific remediation procedures.
2. **Priority overrides** — Override the default priority assignment logic for specific finding types or organisational policies.
3. **Custom phases** — Add custom phases beyond the default three (e.g. "validation", "post-remediation scan").
4. **Effort estimation** — Replace the default effort estimation model with one calibrated to your team's velocity.
5. **Automation integration** — `RemediationAction.automated` and `command` fields are designed for integration with automated remediation runners.

## Examples

### Generating Recommendations

```typescript
import { RecommendationEngine } from './recommendation';

const engine = new RecommendationEngine();

const recommendations = engine.generate(normalizedFindings, riskAssessments);

for (const rec of recommendations) {
  console.log(`[${rec.priority}] ${rec.title}`);
  console.log(`  Finding: ${rec.findingId}`);
  console.log(`  Effort: ${rec.estimatedEffort}`);
  console.log(`  Actions: ${rec.actions.length}`);
  for (const action of rec.actions) {
    console.log(`    - ${action.title} (${action.type}, automated: ${action.automated})`);
  }
}
```

### Creating a Remediation Plan

```typescript
const recommendations = engine.generate(normalizedFindings, riskAssessments);
const plan = engine.createPlan(recommendations);

console.log(`Remediation Plan: ${plan.totalActions} actions, ~${plan.estimatedTotalEffort}`);
console.log(`Critical path: ${plan.criticalPathLength} steps`);

for (const phase of plan.phases) {
  console.log(`\n${phase.name.toUpperCase()} (${phase.timeframe})`);
  console.log(`  ${phase.actionCount} actions, ${phase.recommendations.length} recommendations`);
  for (const rec of phase.recommendations) {
    console.log(`  [${rec.priority}] ${rec.title} — ${rec.estimatedEffort}`);
  }
}
```

### Filtering by Priority

```typescript
// Focus on critical and high priority only
const criticalRecs = recommendations.filter(
  r => r.priority === RecommendationPriority.Critical || r.priority === RecommendationPriority.High
);

const criticalPlan = engine.createPlan(criticalRecs);
console.log(`Critical plan: ${criticalPlan.totalActions} actions`);
```

### Working with Automated Actions

```typescript
// Extract all automated actions for a CI/CD pipeline
const automatedActions = recommendations.flatMap(rec =>
  rec.actions.filter(a => a.automated)
);

for (const action of automatedActions) {
  console.log(`Auto: ${action.title}`);
  if (action.command) {
    console.log(`  Command: ${action.command}`);
  }
  console.log(`  Verify: ${action.verification}`);
}
```

### Tracking Recommendation Status

```typescript
// Simulate status updates
recommendations[0].status = RecommendationStatus.InProgress;
recommendations[1].status = RecommendationStatus.Completed;
recommendations[2].status = RecommendationStatus.Accepted;

const pending = recommendations.filter(r => r.status === RecommendationStatus.Pending);
const done = recommendations.filter(r => r.status === RecommendationStatus.Completed);

console.log(`${done.length}/${recommendations.length} recommendations completed`);
console.log(`${pending.length} still pending`);
```

## Performance Notes

| Aspect | Detail |
|--------|--------|
| **Time complexity** | `generate`: O(n) for n findings. `createPlan`: O(n log n) for dependency resolution and sorting. |
| **Throughput** | ~30 000 recommendations/sec for `generate`. Plan creation is I/O-free and equally fast. |
| **Memory** | Each `Recommendation` with 3–5 actions is ~2 KB. 10 000 recommendations ≈ 20 MB. |
| **Dependency resolution** — Topological sort handles most plans in O(n). Cyclic dependencies are detected and broken automatically. |
| **Template catalogue** — The action template catalogue is loaded once at construction. Large catalogues (> 500 templates) add negligible latency. |
